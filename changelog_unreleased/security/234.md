- Closed a sensitive-file guard evasion: the Edit/Write `.env`/`*.pem`/`credentials*` check matched only the **lexical** basename of the raw target, so a symlink with an innocuous name pointing at a sensitive file (e.g. `ln -s ~/.ssh/id_rsa ./innocent` then editing `./innocent`) wrote through the link unblocked. When the final path component is a symlink the check now also matches the `python3`-realpath-resolved basename (blocks if either the lexical or resolved name is sensitive); `python3` absent/unresolvable falls back to lexical-only (fail-soft). TOCTOU and hardlinks remain out of scope. (#234)
