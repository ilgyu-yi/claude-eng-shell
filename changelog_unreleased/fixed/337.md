- The dir-mode post-merge workflow now flags the SPEC §10.5 bare-`Refs`-to-default anti-pattern: when a PR merges to the live default branch with an empty closing set but a directive-scoped `Refs #N` in its body, it posts a single idempotent warning comment on the PR naming the lost reflection and the recovery. Warn-only, fail-soft, and — being merge-triggered rather than a `gh pr merge` hook — it catches human GitHub-UI merges too (the dominant #92 failure path). (#337)
