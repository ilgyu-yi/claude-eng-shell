Fixed a token-boundary over-match in the `label-parent-consistency` matcher (audit Theme C, EI of Directive #275): its label regex terminated the type-token alternation with `([^a-z]|$)`, but `-` and `_` are `[^a-z]`, so a custom label that merely *starts with* a type token (`--add-label directive-foo`, `task_old`) over-matched the bare token and falsely tripped the matcher (e.g. blocking a `directive-foo` relabel as if it were the `directive` type-key). The terminator is now a true label boundary — comma / whitespace / quote / end — so suffixed labels no longer over-match, while the bare token, comma-joined lists, and the already-excluded letter-suffix case (`executionish`) are unaffected. The sibling `is_directive_issue`/`is_initiative_issue` label greps were already comma-anchored (`(^|,)directive(,|$)`) and needed no change.
