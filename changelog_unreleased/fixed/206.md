- Fixed the SPEC §7 escape hatch being non-functional through the live Claude Code Bash tool: the leading `SKIP_HOOKS=…` env-prefix is consumed by the harness as the subprocess environment and never reaches the hook, so matchers blocked even when an escape was requested. Added a **trailing-sentinel** form — `<command>  # claude-eng:skip=<category>[,…] reason=<why>` — that survives the harness (a `#`-comment stays in the command and is ignored by the executed shell) and is the supported in-harness escape. It is parsed from the raw command before normalization, namespaced so an ordinary comment isn't mistaken for an escape, one-shot, and audited via the existing escape path; the leading env-prefix form still works where it arrives in-string and wins if both are present.
